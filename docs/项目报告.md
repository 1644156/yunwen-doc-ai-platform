## 第一部分：项目概述与背景

---

### 1.1 项目定位

yunwen 是一个基于 RAG (Retrieval-Augmented Generation) 技术的**智能知识库系统**，为设备产品文档提供智能问答能力。

**核心价值：**
- 将分散的 PDF 设备手册自动结构化并入库
- 支持自然语言问答，快速定位产品信息
- 多路检索融合 + CRAG 纠错，保证答案准确性

**典型应用场景：**
- 企业内部产品文档智能客服
- 员工设备使用问答助手
- 技术支持知识库检索

---

### 1.2 技术栈总览

| 层级 | 技术选型 | 说明 |
|------|----------|------|
| **协议层** | FastMCP | 提供 MCP 服务接口 |
| **编排层** | LangGraph | DAG 任务流编排 |
| **LLM 层** | Qwen3.5-plus (阿里云 DashScope) | 大语言模型生成答案 |
| **Embedding** | BGE-M3 (FlagEmbedding) | 混合向量生成 |
| **Reranker** | BAAI/bge-reranker-v2-m3 | 重排序提升精度 |
| **向量库** | Milvus | 高维向量检索 |
| **图数据库** | Neo4j | 知识图谱存储 |
| **文档存储** | MongoDB | 文档元数据与历史 |
| **对象存储** | MinIO | 图片等二进制文件 |
| **PDF解析** | MinerU | PDF 转 Markdown |
| **Python** | 3.12+ | PyTorch CUDA 12.6 |

## 2.1 MCP 服务层

MCP (Model Context Protocol) 是系统的对外接口层，基于 **FastMCP** 框架实现。

### 2.1.1 服务启动

**入口文件**: `app/mcp_server.py`

```python
# 核心启动配置
mcp = FastMCP("yunwen_mcp", lifespan=lifespan)
```

**服务地址**: `http://127.0.0.1:9100/mcp`

### 2.1.2 生命周期管理

```python
@asynccontextmanager
async def lifespan(server: FastMCP):
    """预热：启动时加载模型和连接"""
    # 1. 预加载 BGE-M3 向量模型
    get_bge_m3_ef()
    
    # 2. 预加载重排序模型
    get_reranker_model()
    
    # 3. 预连接 Milvus
    get_milvus_client()
    
    yield {}  # 服务运行中
    
    # 关闭时清理资源（可选）
```

#### 查询工具核心逻辑

```python
@mcp.tool()
async def yunwen_query_knowledge_base(
    query: str,
    session_id: str = "",
    item_names: list[str] | None = None,
    enable_routing: bool = True,
    enable_crag: bool = True,
) -> dict:
    # 1. 参数校验
    query = query.strip()
    if not query:
        return {"success": False, "error": "query must not be empty"}
    
    # 2. 生成会话ID
    session_id = session_id or str(uuid.uuid4())
    
    # 3. 创建初始状态
    state = create_query_default_state(
        session_id=session_id,
        original_query=query,
        item_names=item_names or [],
        is_stream=False,
        config={"enable_routing": enable_routing, "enable_crag": enable_crag},
    )
    
    # 4. 调用 LangGraph 查询流程
    result = await asyncio.wait_for(
        asyncio.to_thread(query_app.invoke, state),
        timeout=60.0  # 60秒超时
    )
    
    # 5. 格式化返回
    return {
        "success": True,
        "answer": result.get("answer", ""),
        "session_id": session_id,
        "citations": citations,
        "image_urls": result.get("image_urls", []),
        "query_type": result.get("query_type", ""),
        "crag_decision": result.get("crag_decision", ""),
    }
```

```python
def node_entry(state: ImportGraphState) -> ImportGraphState:
    # 1. 校验必要参数
    if not state.get("file_path"):
        raise ValueError("file_path 不能为空")
    
    # 2. 读取文件内容
    file_path = state["file_path"]
    
    # 3. 根据 task_type 确定下一步
    task_type = state.get("task_type", "pdf")
    if task_type == "pdf":
        next_node = "node_pdf_to_md"
    elif task_type == "md":
        next_node = "node_md_img"  # 跳过 PDF 转换
    else:
        next_node = "node_bge_embedding"  # 直接向量入库
    
    return {"next_node": next_node}
```

#### Node 2: node_pdf_to_md (PDF 转 Markdown)

**职责**: 调用 MinerU 将 PDF 转换为 Markdown

```python
def node_pdf_to_md(state: ImportGraphState) -> ImportGraphState:
    """使用 MinerU 将 PDF 转为 Markdown"""
    
    from magic_pdf.data.data_reader_writer import FileBasedDataReader
    from magic_pdf.data.dataset import PymuDocDataset
    
    file_path = state["file_path"]
    reader = FileBasedDataReader("")
    
    # 1. 读取 PDF
    if file_path.startswith("http"):
        pdf_bytes = reader.read(file_path)
    else:
        with open(file_path, "rb") as f:
            pdf_bytes = f.read()
    
    # 2. 调用 MinerU 解析
    dataset = PymuDocDataset(pdf_bytes)
    pipe = dataset.fit_inferctl("full")
    
    # 3. 输出 Markdown
    md_content = pipe.pipe_mk_markdown(
        global_info={"file_name": os.path.basename(file_path)}
    )
    
    return {"md_content": md_content}
```

#### Node 3: node_md_img (Markdown 图片处理)

```python
def node_md_img(state: ImportGraphState) -> ImportGraphState:
    """从 Markdown 中提取图片并上传"""
    
    import re
    
    md_content = state["md_content"]
    images = []
    
    # 1. 提取图片 Markdown 语法
    # ![alt](url) 或 ![alt](local_path)
    img_pattern = r'!\[([^\]]*)\]\(([^\)]+)\)'
    matches = re.findall(img_pattern, md_content)
    
    for alt_text, img_path in matches:
        # 2. 上传到 MinIO
        uploaded_url = minio_client.upload_image(img_path)
        
        # 3. 替换为新的 URL
        md_content = md_content.replace(img_path, uploaded_url)
        
        images.append({
            "alt": alt_text,
            "url": uploaded_url,
            "original_path": img_path
        })
    
    return {
        "md_content": md_content,
        "images": images
    }
```

#### Node 4: node_document_split (文档分块)

**职责**: 将长文档切分为适合检索的小块

```python
def node_document_split(state: ImportGraphState) -> ImportGraphState:
    """智能文档分块"""
    
    md_content = state["md_content"]
    chunks = []
    
    # 1. 按 Markdown 标题 (# ## ###) 分割
    sections = re.split(r'^#{1,6}\s+', md_content, flags=re.MULTILINE)
    
    for idx, section in enumerate(sections):
        if not section.strip():
            continue
            
        # 2. 进一步按段落或固定长度分割
        # 策略：每个 chunk 不超过 2000 字符
        if len(section) > 2000:
            # 递归拆分大块
            sub_chunks = split_long_text(section, max_length=2000)
            for sub_idx, sub_text in enumerate(sub_chunks):
                chunks.append({
                    "chunk_id": f"{idx}_{sub_idx}",
                    "content": sub_text,
                    "title": extract_title(section),
                    "level": detect_level(section),
                })
        else:
            chunks.append({
                "chunk_id": str(idx),
                "content": section,
                "title": extract_title(section),
                "level": detect_level(section),
            })
    
    # 3. 过滤空块
    chunks = [c for c in chunks if c["content"].strip()]
    
    return {"chunks": chunks}
```

```python
def node_item_name_recognition(state: ImportGraphState) -> ImportGraphState:
    """使用 LLM 识别文档中的商品名称"""
    
    from app.lm.lm_utils import invoke_llm
    from app.core.load_prompt import load_prompt
    
    chunks = state.get("chunks", [])
    if not chunks:
        return state
    
    # 1. 提取前几个 chunk 的标题作为候选
    candidate_titles = [c.get("title", "") for c in chunks[:10] if c.get("title")]
    candidate_text = "\n".join(candidate_titles)
    
    # 2. 加载提示词
    prompt = load_prompt(
        "item_name_recognition",
        titles=candidate_text
    )
    
    # 3. 调用 LLM
    result = invoke_llm(prompt, json_mode=True)
    import json
    item_names = json.loads(result).get("item_names", [])
    
    return {
        "item_name": item_names[0] if item_names else "",
        "item_names": item_names
    }
```

#### Node 6: node_bge_embedding (向量化)

**职责**: 使用 BGE-M3 生成稠密 + 稀疏向量

```python
def node_bge_embedding(state: ImportGraphState) -> ImportGraphState:
    """BGE-M3 混合向量生成"""
    
    from app.lm.embedding_utils import generate_embeddings
    
    chunks = state.get("chunks", [])
    final_chunks = []
    
    # 1. 批量处理（每批 5 条，避免显存爆炸）
    batch_size = 5
    for i in range(0, len(chunks), batch_size):
        batch_items = chunks[i:i + batch_size]
        
        # 2. 构造向量化文本（商品名前置，提高命中率）
        texts = []
        for item in batch_items:
            item_name = item.get("item_name", "")
            content = item.get("content", "")
            # 核心词前置原则
            text = f"商品：{item_name}，内容介绍：{content}"
            texts.append(text)
        
        # 3. 生成混合向量
        result = generate_embeddings(texts)
        
        # 4. 添加向量到 chunk
        for j, chunk in enumerate(batch_items):
            chunk_item = chunk.copy()
            chunk_item["dense_vector"] = result["dense"][j]
            chunk_item["sparse_vector"] = result["sparse"][j]
            final_chunks.append(chunk_item)
    
    return {"chunks": final_chunks}
```

**向量化文本示例**:
```
商品：华为擎云 G540，内容介绍：电源键位于主机正面...
```

### 3.3.3 node_rrf (倒数排名融合)

**RRF (Reciprocal Rank Fusion)**: 将多路检索结果融合排序

```python
def node_rrf(state: QueryGraphState) -> QueryGraphState:
    """RRF 倒数排名融合"""
    
    K = 60  # RRF 常量，通常取 60
    
    # 1. 收集所有检索结果
    all_chunks = []
    
    # 各种检索结果 (已按相关度排序)
    retrieval_sources = [
        ("embedding_chunks", state.get("embedding_chunks", []), 1.0),
        ("hyde_embedding_chunks", state.get("hyde_embedding_chunks", []), 1.0),
        ("bm25_chunks", state.get("bm25_chunks", []), 0.8),
        ("graph_rag_chunks", state.get("graph_rag_chunks", []), 0.7),
    ]
    
    # 2. 构建文档ID映射
    doc_map = {}  # doc_id -> chunk info
    
    for source_name, chunks, weight in retrieval_sources:
        for rank, chunk in enumerate(chunks, start=1):
            # 使用 content hash 作为唯一标识
            doc_id = hashlib.md5(chunk.get("text", "").encode()).hexdigest()
            
            if doc_id not in doc_map:
                doc_map[doc_id] = chunk.copy()
                doc_map[doc_id]["scores"] = {}
                doc_map[doc_id]["sources"] = []
            
            # 累加 RRF 分数
            rrf_score = weight * (1 / (K + rank))
            doc_map[doc_id]["scores"][source_name] = rrf_score
            doc_map[doc_id]["sources"].append(source_name)
    
    # 3. 计算总分并排序
    fused_chunks = []
    for doc_id, chunk in doc_map.items():
        total_score = sum(chunk["scores"].values())
        chunk["rrf_score"] = total_score
        chunk["source"] = "+".join(set(chunk["sources"]))
        fused_chunks.append(chunk)
    
    # 4. 按 RRF 分数降序排序
    fused_chunks.sort(key=lambda x: x["rrf_score"], reverse=True)
    
    # 5. 断崖式截断 (Top-K + 相对降幅 25%)
    final_chunks = []
    top_score = fused_chunks[0]["rrf_score"] if fused_chunks else 0
    for chunk in fused_chunks[:50]:  # 先取前50
        if chunk["rrf_score"] >= top_score * 0.75:  # 相对降幅 25%
            final_chunks.append(chunk)
        elif top_score - chunk["rrf_score"] < 0.5:  # 绝对降幅 0.5
            final_chunks.append(chunk)
        else:
            break
    
    return {"rrf_chunks": final_chunks}
```

**RRF 公式**:
```
Score(doc) = Σ (weight_i / (K + rank_i))
```
- K = 60 (常用常量)
- weight_i = 各检索路径权重
- rank_i = 该文档在第 i 路检索中的排名

### 3.3.5 node_crag (CRAG 检索质量评估)

**CRAG (Corrective RAG)**: 评估检索结果质量，决定后续处理策略

```python
def node_crag(state: QueryGraphState) -> QueryGraphState:
    """CRAG 检索质量评估"""
    
    reranked_docs = state.get("reranked_docs", [])
    
    # 1. 无结果 -> 拒答
    if not reranked_docs:
        return {"crag_decision": "all_fail", "crag_score": 0.0}
    
    # 2. 过滤无效 chunk (内容太短)
    MIN_CONTENT_LENGTH = 20
    valid_docs = [
        doc for doc in reranked_docs 
        if len(doc.get("text", "").strip()) >= MIN_CONTENT_LENGTH
    ]
    
    if not valid_docs:
        return {"crag_decision": "all_fail", "crag_score": 0.0}
    
    # 3. 计算平均分数 (Top-3)
    top_scores = [
        float(doc.get("score", 0.0)) 
        for doc in valid_docs[:3]
        if doc.get("score") is not None
    ]
    avg_score = sum(top_scores) / len(top_scores) if top_scores else 0.0
    
    # 4. 决策阈值
    CRAG_CORRECT_THRESHOLD = 0.5
    CRAG_INCORRECT_THRESHOLD = 0.2
    
    if avg_score >= CRAG_CORRECT_THRESHOLD:
        decision = "correct"
    elif avg_score >= CRAG_INCORRECT_THRESHOLD:
        decision = "ambiguous"
    else:
        decision = "incorrect"
    
    return {
        "crag_decision": decision,
        "crag_score": avg_score,
    }
```

**CRAG 决策逻辑**:
| 决策 | 分数阈值 | 后续处理 |
|------|----------|----------|
| **correct** | ≥ 0.5 | 直接生成答案 |
| **ambiguous** | 0.2 - 0.5 | 补充检索 (Web Search / Graph) |
| **incorrect** | < 0.2 | 改写查询重试 (最多1次) |
| **all_fail** | - | 拒答，返回友好提示 |

```python
    # 根据问题类型返回不同的拒答文案
    if "如何" in query or "怎么" in query:

    elif "是否" in query or "能不能" in query:
        answer = "抱歉，当前知识库中没有找到与您问题相关的明确答案。建议您咨询产品官方支持。"
    else:
        answer = "抱歉，我无法回答这个问题。知识库中没有找到相关信息。"
    
    return {
        "answer": answer,
        "citations": [],
        "crag_decision": "all_fail"
    }
```

## 4.6 熔断器 (Circuit Breaker)

**文件**: `app/utils/circuit_breaker.py`

```python
import time
from enum import Enum
from threading import Lock

class CircuitState(Enum):
    CLOSED = "closed"      # 正常
    OPEN = "open"          # 熔断
    HALF_OPEN = "half_open"  # 半开

class CircuitBreaker:
    """熔断器实现"""
    
    def __init__(
        self,
        failure_threshold: int = 5,
        recovery_timeout: int = 60,
        success_threshold: int = 2
    ):
        self.failure_threshold = failure_threshold
        self.recovery_timeout = recovery_timeout
        self.success_threshold = success_threshold
        
        self.failure_count = 0
        self.success_count = 0
        self.last_failure_time = None
        self.state = CircuitState.CLOSED
        self._lock = Lock()
    
    def call(self, func, *args, **kwargs):
        """执行函数，带熔断保护"""
        
        with self._lock:
            # 检查状态
            if self.state == CircuitState.OPEN:
                if time.time() - self.last_failure_time > self.recovery_timeout:
                    self.state = CircuitState.HALF_OPEN
                else:
                    raise CircuitOpenError("Circuit breaker is OPEN")
            
            try:
                result = func(*args, **kwargs)
                
                # 成功后处理
                if self.state == CircuitState.HALF_OPEN:
                    self.success_count += 1
                    if self.success_count >= self.success_threshold:
                        self.state = CircuitState.CLOSED
                        self.failure_count = 0
                        self.success_count = 0
                
                return result
                
            except Exception as e:
                self.failure_count += 1
                self.last_failure_time = time.time()
                
                if self.failure_count >= self.failure_threshold:
                    self.state = CircuitState.OPEN
                
                raise e


class CircuitOpenError(Exception):
    """熔断器打开异常"""
    pass


# 全局熔断器实例

    failure_threshold=3,
    recovery_timeout=30,
    success_threshold=1
)
```

```python
def load_prompt(name: str, **kwargs) -> str:
    """

        rendered_prompt = raw_prompt.format(**kwargs)
        return rendered_prompt
    
    return raw_prompt
```

|------|------|------|
| **answer_relevancy** | 答案与问题的相关性 | 0-1 |
| **faithfulness** | 答案是否忠实于检索文档 | 0-1 |
| **context_precision** | 检索文档与问题的相关性 | 0-1 |
| **context_recall** | 检索文档覆盖答案的程度 | 0-1 |

### 5.3.2 RAGAS 评估器

**文件**: `app/eval/ragas_evaluator.py`

```python
from ragas import evaluate
from ragas.metrics import (
    answer_relevancy,
    faithfulness,
    context_precision,
    context_recall,
)

def evaluate_rag_response(
    query: str,
    answer: str,
    contexts: list,
    ground_truth: str = None
) -> dict:
    """评估单个 RAG 响应"""
    
    # 构建评估数据
    eval_data = {
        "question": [query],
        "answer": [answer],
        "contexts": [contexts],
    }
    
    # 如果有标准答案，添加 answer_correctness
    if ground_truth:
        eval_data["ground_truth"] = [ground_truth]
        metrics = [
            answer_relevancy,
            faithfulness,
            context_precision,
            context_recall,
            answer_correctness,
        ]
    else:
        metrics = [
            answer_relevancy,
            faithfulness,
            context_precision,
            context_recall,
        ]
    
    # 执行评估
    result = evaluate(
        data=eval_data,
        metrics=metrics,
    )
    
    return result.to_pandas().to_dict(orient="records")[0]
```

```python
import json
```

### 5.3.4 实验管理器

**文件**: `app/eval/experiment_manager.py`

```python
import json
from datetime import datetime
from pathlib import Path

class ExperimentManager:
    """实验管理器 - 记录和管理评估实验"""
    
    def __init__(self, base_dir: str = "experiments"):
        self.base_dir = Path(base_dir)
        self.base_dir.mkdir(exist_ok=True)
    
    def create_experiment(
        self,
        name: str,
        config: dict
    ) -> str:
        """创建新实验"""
        
        timestamp = datetime.now().strftime("%Y%m%d_%H%M%S")
        exp_dir = self.base_dir / f"{name}_{timestamp}"
        exp_dir.mkdir()
        
        # 保存配置
        with open(exp_dir / "config.json", "w") as f:
            json.dump(config, f, indent=2)
        
        return str(exp_dir)
    
    def record_result(
        self,
        exp_dir: str,
        query: str,
        metrics: dict
    ):
        """记录单次评估结果"""
        
        result_file = Path(exp_dir) / "results.jsonl"
        
        with open(result_file, "a", encoding="utf-8") as f:
            f.write(json.dumps({
                "query": query,
                "metrics": metrics,
                "timestamp": datetime.now().isoformat()
            }, ensure_ascii=False) + "\n")
    
    def summarize(self, exp_dir: str) -> dict:
        """汇总实验结果"""
        
        result_file = Path(exp_dir) / "results.jsonl"
        
        results = []
        with open(result_file, "r") as f:
            for line in f:
                results.append(json.loads(line))
        
        # 计算各指标均值
        metric_names = set()
        for r in results:
            metric_names.update(r["metrics"].keys())
        
        summary = {}
        for metric in metric_names:
            values = [r["metrics"][metric] for r in results if metric in r["metrics"]]
            summary[metric] = {
                "mean": sum(values) / len(values) if values else 0,
                "min": min(values) if values else 0,
                "max": max(values) if values else 0,
                "count": len(values)
            }
        
        return summary
```

```python
def get_optimal_batch_size(text_length: int, available_memory: int = 8 * 1024**3):
    """根据文本长度动态调整批次大小"""
    
    # 估算每条文本的显存占用
    per_text_memory = text_length * 4 * 1024  # 约 4KB per 1K chars
    
    # 安全边界 80%
    safe_memory = available_memory * 0.8
    
    batch_size = int(safe_memory / per_text_memory)
    
    return max(1, min(batch_size, 32))  # 1-32 之间
```

```python
@app.middleware("http")

    return await call_next(request)
```

## 7.3 数据流转总结

### 7.3.1 导入数据流

```
PDF 文档
    │
    ▼
[MinerU] PDF 解析
    │
    ▼
Markdown + 图片
    │
    ├──▶ [MinIO] 图片上传
    │
    ▼
[文档切块] 智能分块
    │
    ▼
[LLM] 商品名称识别
    │
    ▼
[BGE-M3] 混合向量生成
    │
    │  dense_vector (1024维)
    │  sparse_vector (稀疏字典)
    ▼
[Milvus] 向量入库
    │
    ▼
kb_chunks 集合
```

### 7.3.2 查询数据流

```
用户 Query
    │
    ▼
[LLM] 查询意图分类
    │ factual / reasoning / temporal / ambiguous
    ▼
并行多路检索
    │
    ├── [Embedding] 稠密向量
    ├── [HyDE] 假设答案
    ├── [BM25] 稀疏检索
    ├── [Graph] 知识图谱
    └── [Web] MCP 联网
    │
    ▼
[RRF] 融合排序
    │ score = Σ weight / (60 + rank)
    ▼
[Rerank] BGE 重排序
    │
    ▼
[CRAG] 质量评估
    │ correct / ambiguous / incorrect / all_fail
    ▼
    ├── correct ──▶ [LLM] 生成答案
    ├── ambiguous ──▶ [补充检索] ──▶ 重新生成
    ├── incorrect ──▶ [改写查询] ──▶ 重新检索
    └── all_fail ──▶ [拒答] 返回友好提示
    │
    ▼
[MongoDB] 保存历史
    │
    ▼
返回答案 + 引用
```

## 7.8 总结

yunwen 系统通过以下设计实现了高质量的智能问答：

1. **多路检索融合**: 稠密向量、稀疏检索、知识图谱、HyDE、Web Search 各展所长

4. **模块化架构**: LangGraph DAG 编排，各节点职责清晰，易于扩展
5. **完整评估体系**: RAGAS + 金标集 + 实验管理，数据驱动优化

该系统可作为一个**企业级 RAG 知识库的参考实现**，在文档智能客服、技术支持问答、产品手册检索等场景具有广泛的应用价值。
